# 1. commit the built dist/

- Status: accepted
- Date: 2026-06-20

## Context
Most of `dist/` is generated by tsup, but it also holds hand-authored files (`recipes.css`,
`theme-toggle.js`) and the token-generated CSS. Consumers import these directly through the package
`exports`, so dist is not purely a build artifact.

## Decision
Track `dist/` in git and ship it with the package (`files` includes it). tsup runs with `clean: false`
so the build never wipes the hand-authored parts. `"use client"` is prepended after the build by
`scripts/add-use-client.mjs`, since esbuild drops the module-level directive.

## Consequences
Rebuild and commit `dist/` when components or tokens change. Do not delete it as build output. The
post-build use-client step is required for the entry to act as a client boundary in consumer apps.
